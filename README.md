# 🔎오픈 미션

--- 
### 📢 오픈 미션 소개
> - 오픈 미션은 2주 동안 자신이 도전하고 싶은 주제를 정해 직접 계획을 세우고 실행해보는 프리코스 자율 챌린지입니다.<br>
> - 스스로 목표를 세우고 해결 과정을 경험하며 성장하는 것을 목표로 합니다.

---
> ## 1️⃣ 첫 번째 미션 소개
>> <b>주제</b> : **Schedule 도메인에 대해서 N + 1 문제 해결하기**<br>
>
>> <b>기간</b> : 2025.11.04 ~ 2025.11.10
> 
>> <b>미션 내용</b><br>
>> - `지연 로딩(Lazy Loading)`과 `즉시 로딩(Eager Loading)`에 대해서 학습합니다.<br>
>> - `N + 1 문제`에 대해서 학습합니다.<br>
>> - 학습한 내용을 바탕으로, `Schedule 도메인`에서 발생하는 `N + 1 문제`를 타당한 이유와 함께 찾습니다.
>> - `N + 1 문제`를 해결할 수 있는 방법 중, 가장 적합한 방식을 타당한 이유와 함께 선정합니다.
>
> ### 📌 주제 선정 이유 <br>
>> - `N + 1 문제`는 유명한 문제이지만, `지연 로딩` 전략과 `N + 1 문제`의 관계를 잘 이해하지 못하고 있었습니다.
>> - 프로젝트 내 엔티티를 살펴보니, `@OneToMany`, `@ManyToOne`, `@OneToOne` 과 같은 연관관계를 설정하고, `Lazy 로딩 전략`을 함께 사용하고 있었습니다.<br> 
>> - 이러한 연관 관계에서 `N + 1 문제`가 발생할 수 있다고 판단했고, 이를 해결하는 방법을 배워보고자 했습니다.<br>
>

### 커밋 메시지 컨벤션 정리
| 타입    | 설명 |
|-------|------|
| feat  | 새로운 기능 추가 |
| fix   | 버그 수정 |
| docs  | 문서 수정 |
| style | 코드 스타일 수정 (로직 변경 없음) |
| refactor | 리팩터링 (동작 변화 없이 구조 개선) |
| test  | 테스트 코드 추가/수정 |
| chore | 빌드, 설정, 유지보수 작업 |
